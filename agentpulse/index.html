Features

Alerting and cost analytics built in from day one

From per-product spend breakdowns to instant alert rules — a full observability stack with no SDK or instrumentation changes.

Cost by Product

Bar chart of USD spend per product with % share of total. Click any product to filter the entire dashboard — know exactly what's burning budget before your bill arrives.

Alert Rules

5 configurable metrics: error_rate, success_rate, avg_latency_ms, failed_runs, cost_usd. Set gt/lt thresholds with rolling windows — fires to Slack, webhook, or email.

Webhook Ingestion

POST /v1/runs with a Bearer secret. No SDK, no library, no agent code changes. Includes status, duration, steps, tokens, and cost.

Multi-tenant Orgs

Full org isolation — each organisation sees only its own agents, runs, and alerts. Invite team members, assign admin/member roles, and manage products per org.

Execution Traces

Step-by-step breakdown of every run — LLM calls, tool calls, retrievals, and reasoning steps with per-step timing, token usage, and error context.

Live Dashboard

Real-time run volume charts, agent fleet health, success rate, avg latency, and active agent count. 24h / 7d / 30d period selector across all views.

How it works

Up and running in minutes

1

Deploy

Clone the repo and run make deploy-init on your server. Caddy handles HTTPS automatically.

2

Send runs

Register an agent in the dashboard to get a Bearer secret, then POST one curl after each agent run. No SDK, no library.

3

Alert & analyse

Set alert rules on error rate, cost, or latency. Watch cost-by-product on the dashboard. Get notified on Slack or webhook the moment a threshold is breached.
