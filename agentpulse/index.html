Features

Everything you need to monitor AI agents

From high-level fleet health to per-step execution traces — without any SDK or instrumentation changes.

Live Dashboard

Active runs, success rate, avg latency, and throughput — refreshed live. 30-minute interval run volume and latency trend charts.

Agent Fleet View

Per-agent cards with status, success rate, latency, and run count. Filter by active / degraded / inactive. Click through to agent-level detail.

Runs List

All executions in one table — status, duration, trigger source, and token cost. Filter by status, search by agent name or run ID.

Execution Traces

Step-by-step breakdown of every run — LLM calls, tool calls, retrievals, and reasoning steps with per-step timing and token usage.

Token Cost Tracking

Input and output token counts plus USD cost per run and per step. Understand which agents and which steps are driving your LLM spend.

Auth & Multi-user

Protected routes with login, logout, and change-password. Self-hostable with Docker + Caddy — full control of your observability data.

How it works

Up and running in minutes

1

Deploy

Clone the repo and run make deploy-init on your server. Caddy handles HTTPS automatically.

2

Register agents

Point your agents at the AgentPulse webhook endpoint or use the SDK to push run events. No code changes required for HTTP-based agents.

3

Monitor

Open the dashboard and watch runs arrive in real time. Drill into any run to see the full execution trace and token breakdown.
