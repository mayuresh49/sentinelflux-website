Everything your auditor needs, already in your test data

No separate compliance tooling. SentinelFlux maps what you already run to the controls that matter.

Framework-Mapped Controls

Test runs are automatically mapped to the relevant controls — SOC 2 CC6/CC7/CC8, ISO 27001 A.14.2/A.12.6, or GDPR Art.25/Art.32. Every control shows run count, pass rate, and coverage status.

  • ISO 27001:2022

Complete Audit PDF

The generated PDF contains everything an auditor expects: executive summary, control coverage table, timestamped run evidence log, human approval records, AI agent activity, and a signed declaration.

  • Timestamped run evidence log
  • Human approval records

No new workflow. Run your tests as usual — SentinelFlux collects the evidence automatically. When audit time comes, select a date range, pick a framework, and download the PDF. Done in under a minute.

  • No manual evidence collection

What's inside the PDF

Every section is structured to satisfy an auditor's evidence request — not just pretty charts.

1
Cover + Executive Summary
Product, framework, evidence period, total runs, pass rate, domains covered, sign-off status at a glance.
2
Framework Control Coverage
Each control mapped to test domains, with run count, total tests, and pass rate. Green = covered, amber = no data.
3
Test Coverage by Domain
Per-domain table: runs, total tests, pass/fail counts, pass rate, total duration — broken out by API, Web, Security, Performance, Accessibility.
4
Test Run Evidence Log
Every test run in the period: run ID, domain, module, trigger type (manual/schedule), status, pass rate, UTC timestamp.
5
Human Approval Records
All human-in-the-loop decisions (approvals/rejections) made during the period — type, title, decision, resolver, and timestamp.
6
Sign-off Declaration
Named declaration block with role, date, and signature field — ready to print and countersign for your auditor package.
