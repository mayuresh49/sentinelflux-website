AgentPulse Setup

AgentPulse is real-time observability for AI agents — cost-per-product, run traces, and alerting. This guide is the reference starting point for instrumenting any agent codebase. SentinelFlux ships with a one-line integration that monitors every agent at once.

How it works

AgentPulse is push-based. Registering an agent only mints a webhook secret — nothing is monitored until your agent code sends a run event to POST /v1/runs after each execution. The dashboard shows each agent as “awaiting run data” until the first event arrives.

The single integration point

Every SentinelFlux agent subclasses BaseAgent and implements run(**kwargs) -> dict. That shared base class is the one place to instrument: a __init_subclass__ hook auto-wraps each subclass’s run() as the class is defined — so all agents emit run events with zero changes to the agent classes or their callers. The wrapper times the call, marks the run failed on exception (and re-raises), and tags the event with the instance’s name.

# ai/agents/base_agent.py
class BaseAgent:
    name: str = "base"

    # Auto-wrap every subclass's run() with AgentPulse telemetry (no-op unless configured).
    def __init_subclass__(cls, **kwargs):
        super().__init_subclass__(**kwargs)
        if "run" in cls.__dict__:
            from sentinelflux.agentpulse import instrument_run
            cls.run = instrument_run(cls.__dict__["run"])

Why this scales: __init_subclass__ fires for subclasses at any depth, so a new agent — or one that inherits run() from another agent — is covered automatically. Attribution is by self.name at call time, so inherited run() still reports under the right agent.

The wrapper itself is small and never lets telemetry surface in the agent:

# sentinelflux/agentpulse.py
def instrument_run(fn):
    @functools.wraps(fn)
    def wrapper(self, *args, **kwargs):
        started = datetime.now(timezone.utc)
        t0 = time.monotonic()
        status, error = "success", None
        try:
            return fn(self, *args, **kwargs)
        except Exception as e:
            status, error = "failed", str(e)[:2000]
            raise
        finally:
            record_run(getattr(self, "name", "unknown"), status,
                       started.isoformat(), int((time.monotonic() - t0) * 1000), error)
    return wrapper

1 · Register your agents

Each agent maps to one AgentPulse agent (so the dashboard breaks cost and reliability down per agent). The registration script logs in, creates a VigilQA product, registers every agent, and writes their webhook secrets to a gitignored file. It is idempotent — re-running only creates what’s missing.

AGENTPULSE_URL=https://agentpulse.sentinelflux.in \
AGENTPULSE_EMAIL=you@example.com \
AGENTPULSE_PASSWORD=•••••• \
python scripts/register_agentpulse.py
# → wrote 19 secrets → config/agentpulse.secrets.json

Plan limits: the Free plan allows 5 agents. SentinelFlux registers ~19, so the AgentPulse org must be on Pro (100 agents) before running the script, or registration stops at the limit with a 402.

2 · Configure

The client reads the AgentPulse base URL and per-agent secrets from either source. If neither is present, telemetry is a silent no-op — agents run exactly as before.

  • AGENTPULSE_URL env + config/agentpulse.secrets.json (written by the script), or
  • AGENTPULSE_SECRETS env pointing at that JSON file.
// config/agentpulse.secrets.json  (gitignored — contains secrets)
{
  "base_url": "https://agentpulse.sentinelflux.in",
  "agents": {
    "api_test_synthesis": "whsec_…",
    "flaky_detector": "whsec_…",
    "result_analyzer": "whsec_…"
  }
}

3 · Verify

  1. Run any agent (e.g. a generation or post-run agent) as you normally would.
  2. Open the AgentPulse dashboard — the agent flips from “awaiting run data” to live, with run count, success rate, and latency.
  3. Force a failure and confirm the run shows failed with the error message.

What gets sent

One run event per agent execution:

FieldSource
statussuccess, or failed if run() raised
startedAt / durationMsmeasured around the wrapped call
errorexception message (truncated) on failure
triggeredByapi

Token counts and cost are not yet sent — per-agent token usage is already tracked via usage_scope(), so enriching events with inputTokens/outputTokens/costUsd is a follow-up that lights up AgentPulse’s cost-per-product view.

Troubleshooting

  • Agent stays “awaiting run data” — the run completed but no event was sent. Check that AGENTPULSE_URL is set and the agent’s name exists in the secrets file.
  • Registration stops early with 402 — the org hit its plan’s agent limit; upgrade to Pro and re-run (idempotent).
  • 401 on ingest — the webhook secret was regenerated in AgentPulse; re-run the registration script to refresh the secrets file.
  • Nothing breaks but no data — telemetry is intentionally fire-and-forget and swallows all errors, so a misconfigured URL never fails a run. Verify connectivity to /v1/runs manually with curl.
