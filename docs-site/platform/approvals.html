Approvals

The Approvals queue is where AI-generated artifacts wait for human review before being committed to your test suite.

For the conceptual explanation of the approvals workflow, see Core Concepts — Approvals. This page covers the platform-level Approvals view and its filters.

Approvals queue view

Navigate to Approvals in the left sidebar. The queue is split into tabs:

  • Pending — items awaiting review (your primary action queue)
  • From Agents — items generated by post-run agents (LocatorHealer patches, regeneration requests)
  • Approved — historical log of approved items
  • Rejected — items you've rejected, with your rejection comments

Filtering the queue

Filter pending items by:

  • Type — Script, Document, Locator Patch, Regeneration Request
  • Domain — filter to a specific functional area
  • Priority — scenario priority (high, medium, low)
  • Submitted by — agent name or team member
  • Age — how long the item has been waiting

Badges and indicators

  • Modified badge — item was edited by a team member before approval; the edit is logged
  • High priority badge — scenario has priority: high in the KB
  • Confidence score — shown on LocatorHealer patches (colour-coded: green ≥0.9, yellow 0.7–0.89, red <0.7)
  • Retry count — shows how many times an agent has retried generating this item
