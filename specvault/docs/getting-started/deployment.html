SpecVault Docs

Deployment

Deploy SpecVault on your own infrastructure using Docker or a direct Python setup. Supports any Linux server, GCP, or AWS.

Docker deployment (recommended)

The fastest path to a running instance. Clone the repo, configure your .env, and start:

git clone git@github.com:your-org/specvault.git
cd specvault
cp .env.example .env
# Edit .env with your values (see Environment Variables below)
docker compose up -d

The app starts on port 8000 by default. Put Caddy or Nginx in front for TLS.

Caddy for TLS

Caddy is the recommended reverse proxy because it handles certificate renewal automatically via Let's Encrypt:

# Caddyfile
specvault.example.com {
    reverse_proxy localhost:8000
}
caddy start --config /etc/caddy/Caddyfile

Point your domain's A record at the server IP, and Caddy will obtain and renew the TLS certificate automatically.

GCP deployment

  1. Create a GCP Compute Engine VM (e2-micro or larger). Choose Ubuntu 22.04.
  2. SSH into the VM and install Docker: curl -fsSL https://get.docker.com | sh
  3. Clone the SpecVault repo and configure .env
  4. Start with docker compose up -d
  5. Install Caddy and configure a Caddyfile pointing at port 8000
  6. Open ports 80 and 443 in the VM's firewall rules

AWS deployment

The process is identical on an EC2 instance. Use a t3.micro or larger. Open ports 80 and 443 in the security group. Use an Elastic IP for a stable address to point your DNS at.

Data volume

SpecVault stores all data in a single SQLite file at data/specvault.db. In Docker Compose, this directory is mounted as a volume:

# docker-compose.yml (relevant section)
volumes:
  - ./data:/app/data

Back up the data/ directory to keep your spec history. A daily cron job copying specvault.db to object storage is sufficient for most deployments.

Environment variables

VariableRequiredDescription
SECRET_KEYYesRandom string used for session signing. Generate with openssl rand -hex 32.
ADMIN_EMAILYesEmail address for the default admin account.
ADMIN_PASSWORDYesPassword for the default admin account. Change after first login.
SMTP_HOSTYesSMTP server hostname for sending alert emails.
SMTP_PORTYesSMTP port (typically 587 for STARTTLS, 465 for SSL).
SMTP_USERYesSMTP authentication username.
SMTP_PASSWORDYesSMTP authentication password.
SMTP_FROMYesFrom address for outgoing alert emails (e.g. specvault@example.com).
DATABASE_PATHNoPath to the SQLite database file. Defaults to data/specvault.db.
BASE_URLNoPublic URL of the SpecVault instance. Used in email alert links.

Security note: Never commit your .env file to version control. Add it to .gitignore. Use a secrets manager (GCP Secret Manager, AWS Secrets Manager) for production deployments.

Direct Python setup (no Docker)

If you prefer running the app directly:

python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env
uvicorn dashboard.app:app --host 0.0.0.0 --port 8000

For production, run uvicorn behind a systemd service so it restarts on failure.
