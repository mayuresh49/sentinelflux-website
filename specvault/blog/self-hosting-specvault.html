Self-Hosting SpecVault on GCP in 10 Minutes

A step-by-step guide to spinning up SpecVault on a GCP VM with Docker and Caddy for TLS — your specs stay on your infrastructure, and the whole stack runs on about $6/month.

SpecVault is designed to self-host. The entire stack is a single Python process, a SQLite database, and a JSON audit log — no Postgres, no Redis, no separate worker processes. A $6/month e2-micro on GCP is more than enough for most teams.

Step 1 — Create a VM

In the GCP console or via gcloud, create an e2-micro instance (or equivalent on AWS/Hetzner/DigitalOcean):

gcloud compute instances create specvault \
  --machine-type=e2-micro \
  --image-family=debian-12 \
  --image-project=debian-cloud \
  --zone=us-central1-a \
  --tags=http-server,https-server

Open ports 80 and 443 in your firewall rules so Caddy can handle TLS certificate provisioning.

Step 2 — Install Docker and clone the repo

sudo apt update && sudo apt install -y docker.io git
sudo systemctl enable docker && sudo systemctl start docker

git clone https://github.com/sentinelflux/specvault.git /opt/specvault
cd /opt/specvault

Step 3 — Configure environment variables

cp .env.example data/.env

Edit data/.env with your settings:

SECRET_KEY=generate-a-long-random-string-here
ADMIN_EMAIL=you@yourcompany.com
ADMIN_PASSWORD=choose-a-strong-password

SMTP_HOST=smtp.sendgrid.net
SMTP_PORT=587
SMTP_USER=apikey
SMTP_PASSWORD=your-sendgrid-api-key
SMTP_FROM=specvault@yourcompany.com
SMTP_TLS=true

For SECRET_KEY, generate one with: python3 -c "import secrets; print(secrets.token_hex(32))"

Step 4 — Build and start the container

docker build -t specvault .
docker run -d \
  --name specvault \
  --restart unless-stopped \
  -p 8000:8000 \
  -v /opt/specvault/data:/app/data \
  --env-file /opt/specvault/data/.env \
  specvault

SpecVault is now running on port 8000. The -v flag mounts the data directory so your SQLite database and audit log survive container restarts and updates.

Step 5 — Add Caddy for TLS

Point your domain's A record at the VM's IP, then install and configure Caddy:

sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update && sudo apt install caddy

Create /etc/caddy/Caddyfile:

specvault.yourcompany.com {
    reverse_proxy localhost:8000
}
sudo systemctl reload caddy

Caddy automatically provisions a Let's Encrypt certificate. Your instance is now live at https://specvault.yourcompany.com.

Step 6 — Log in and register your first service

Open the dashboard, log in with the ADMIN_EMAIL and ADMIN_PASSWORD you set, and follow the Quick Start guide to add your first API.

Backups

SpecVault's entire persistent state lives in two files:

  • data/specvault.db — SQLite database (all services, specs, subscriptions, tokens)
  • data/audit_log.json — append-only audit log

Back these up with a simple cron job:

0 3 * * * cp /opt/specvault/data/specvault.db /opt/specvault/data/backups/specvault-$(date +\%Y\%m\%d).db

Or use gsutil cp to ship daily backups to a GCS bucket.

Updates

cd /opt/specvault
git pull
docker build -t specvault .
docker stop specvault && docker rm specvault
# re-run the docker run command from Step 4

The data directory is mounted outside the container, so updates never touch your database or audit log.
