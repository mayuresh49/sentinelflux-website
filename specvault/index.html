Works in three steps

No new tooling to learn. If you have an OpenAPI spec, you're already set up.

1

Register your API

Add a service — name it after your API, add the team owner and base URL. One time setup.

2

Publish spec versions

Paste your spec URL or upload a JSON/YAML file. Each publish auto-diffs against the previous version.

3

Subscribers get alerted

Consumer teams subscribe and receive email alerts the moment a breaking change is detected — before they're broken.

Everything a platform team needs

Version History

Every spec version stored with endpoint count, publish date, and author. Diff any two versions on demand.

Breaking Change Detection

Automatically flags removed endpoints, deleted required params, new required fields, and removed response fields.

Email Alerts

Consumer teams subscribe per service. Breaking-change alerts land in their inbox before the next deploy ships.

CI/CD Integration

Publish specs via API token from your pipeline. SentinelFlux auto-pushes specs after every successful contract test run.

API Tokens

Generate machine tokens for headless publishing from CI. Tokens are bcrypt-hashed; plain value shown once.

Self-Hostable

Deploy on your own GCP / AWS instance in under 10 minutes. Full data ownership — your specs never leave your infrastructure.
