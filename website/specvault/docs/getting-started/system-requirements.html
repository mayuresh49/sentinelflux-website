System Requirements

Minimum and recommended requirements for running a SpecVault instance.

Server requirements

ComponentMinimumRecommended
CPU1 vCPU2 vCPU
Memory512 MB RAM1 GB RAM
Disk1 GB10 GB (for large spec histories)
OSLinux (any distro)Ubuntu 22.04 LTS

A GCP e2-micro or AWS t3.micro is sufficient for teams with up to 50 services and daily publishes. Scale up if you're storing very large OpenAPI specs or have hundreds of services.

Python

SpecVault requires Python 3.12 or later. Python 3.11 may work but is not tested. Python 3.10 and earlier are not supported.

When using Docker, the correct Python version is included in the container image — no manual installation needed.

Database

SpecVault uses SQLite with WAL mode enabled. No separate database server is required. The database file is created automatically on first startup at the path specified in DATABASE_PATH (default: data/specvault.db).

SQLite with WAL mode handles concurrent reads and writes correctly for SpecVault's access patterns. A PostgreSQL migration is on the roadmap for large enterprise deployments but is not currently supported.

SMTP for notifications

An SMTP server is required for sending breaking-change email alerts to subscribers. SpecVault supports any standard SMTP server:

  • SendGrid — recommended for production. Use SMTP relay on port 587.
  • AWS SES — good option for AWS-hosted deployments.
  • Mailgun — straightforward setup with free tier.
  • Gmail SMTP — acceptable for small teams. Requires an App Password if 2FA is enabled.
  • Self-hosted (Postfix/Exim) — works but requires correct SPF/DKIM configuration to avoid spam filtering.

If SMTP is not configured, SpecVault will still function — breaking changes will be detected and displayed in the dashboard, but no email alerts will be sent to subscribers.

Network requirements

Inbound

  • Port 443 (HTTPS) — for the dashboard and API, accessed by users and CI/CD pipelines
  • Port 80 (HTTP) — used by Caddy for ACME certificate challenges; all traffic is redirected to HTTPS

Outbound

  • Port 443 — required for fetching spec files from remote URLs (when using "Fetch from URL")
  • SMTP port (587 or 465) — required for sending alert emails
  • Port 443 to Let's Encrypt — required for Caddy's automatic TLS certificates

Browser requirements

The SpecVault dashboard works in any modern browser. Tested on:

  • Chrome 120+
  • Firefox 120+
  • Safari 17+
  • Edge 120+

JavaScript must be enabled. No browser extensions are required.
