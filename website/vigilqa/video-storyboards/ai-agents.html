AI Agents

10 agents — 5 generation pipeline + 5 post-run analysis

~5:00
estimated duration
Target: technical leads, decision-makers
Scene 1 — Hook0:00 – 0:20

On Screen

Split screen: left — traditional QA workflow with humans at every step. Right — VigilQA with agents at each step. Agents highlighted one by one.

Voiceover

"VigilQA has ten AI agents working behind the scenes. Five generate and review test scripts from your Knowledge Base — running once per generation cycle. Five analyse every test run automatically — running after every execution. Together, they replace the manual work that keeps most QA teams from scaling."

Scene 2 — Generation agents overview0:20 – 1:00

On Screen

Pipeline diagram: AppExplorer → DocGen → DocReview → ScriptGen → ScriptReview. Each node has a brief label. Arrow shows left-to-right flow with approval gates between DocReview/ScriptGen and ScriptReview/commit.

Voiceover

"The generation pipeline has five agents in sequence. AppExplorer crawls your application and maps every page, selector, and interactive element. DocGen drafts test case documents from your KB scenarios. DocReview checks those documents for accuracy and completeness. ScriptGen converts reviewed documents into executable pytest scripts. ScriptReview validates the scripts before they're committed."

Scene 3 — AppExplorer demo1:00 – 1:40

On Screen

Activities view: AppExplorer run in progress. Show the pages it has discovered, selector maps being built. Show the output — a structured JSON of pages → elements → selectors with stability rankings.

Voiceover

"AppExplorer is the foundation. It browses your application exactly as a user would, discovering every page and capturing selectors for every interactive element. It ranks selectors by stability — preferring data-testid attributes over CSS classes — so the tests ScriptGen produces are as robust as possible from day one."

Scene 4 — Post-run agents overview1:40 – 2:20

On Screen

Test run complete → 5 agents fire simultaneously: ResultAnalyzer, FlakyDetector, RegressionGuard, CoverageGap, LocatorHealer. Each has a status indicator (running → done). Agent activity feed shows their output arriving.

Voiceover

"After every test run, all five analysis agents fire automatically. ResultAnalyzer classifies each failure — real bug, locator issue, or environment flakiness. FlakyDetector scores tests by flakiness across runs. RegressionGuard compares this run to recent history to detect new failures. CoverageGap identifies KB scenarios with no test coverage. LocatorHealer queues healing proposals for broken selectors."

Scene 5 — Agent activities view2:20 – 3:10

On Screen

Navigate to Activities. Show a timeline of agent actions: "ResultAnalyzer: 3 failures classified as real bugs" · "FlakyDetector: 2 tests flagged as flaky (score >0.4)" · "LocatorHealer: 1 heal proposal queued" · "CoverageGap: 4 KB scenarios have no tests". Click through to each finding.

Voiceover

"The Activities view is your agent inbox. After every run, you see a summary of everything the agents found and did. Three real bugs to investigate. Two flaky tests to suppress or fix. One locator heal to approve. Four untested scenarios to prioritise in your next KB update. Everything surfaced without you digging."

Scene 6 — FlakyDetector and RegressionGuard3:10 – 3:50

On Screen

FlakyDetector view: test list with flakiness scores (0–1 scale). One test highlighted at 0.7 — "high flakiness, recommend suppression or fix." Then RegressionGuard: "2 tests that passed in the last 5 runs are now failing — new regression detected."

Voiceover

"FlakyDetector tracks each test's pass/fail pattern across runs. A test that fails 7 times out of 10 gets a high flakiness score — VigilQA will tell you to investigate it rather than treat every failure as a real bug. RegressionGuard takes the opposite view — it watches for tests that have been reliably passing and suddenly fail, which is usually a genuine regression."
