Performance Testing

Response time, throughput, Lighthouse metrics — baselines and regression alerts

~4:00
estimated duration
Target: backend engineers, SREs, product teams
Scene 1 — Hook0:00 – 0:20

On Screen

Chart: API response time over 30 deploys. Flat line at 120ms, then sudden spike to 800ms at deploy 24. Text: "Nobody noticed for 3 days."

Voiceover

"Performance regressions are silent. A deploy that adds a missing database index, or loads an extra library, can double your response time without breaking a single test. By the time you notice, it's in your user's experience and in your metrics. VigilQA's performance module tracks baselines and alerts you when a deploy crosses them."

Scene 2 — What it measures0:20 – 0:55

On Screen

Two columns: API metrics (response time p50/p95/p99, throughput, error rate) and Frontend metrics (Lighthouse scores: Performance, LCP, FCP, CLS, TBT)

Voiceover

"VigilQA's performance module covers two dimensions. For APIs: response time percentiles — p50, p95, p99 — plus throughput and error rate under load. For web pages: Lighthouse-based scores covering overall performance, Largest Contentful Paint, Cumulative Layout Shift, and Total Blocking Time. Both run in your CI environment against your real application."

Scene 3 — KB scenario and thresholds0:55 – 1:35

On Screen

KB scenario with module: performance. Show: target endpoints list, thresholds (p95 ≤ 300ms, error rate ≤ 1%, Lighthouse Performance ≥ 85). Show "baseline" toggle — first run establishes baseline.

Voiceover

"You define your performance thresholds in the KB scenario — what p95 response time is acceptable, what error rate is tolerable, what Lighthouse score you require. The first run establishes the baseline. From the second run onwards, VigilQA checks each metric against the threshold and flags any regression. You set the budget; VigilQA enforces it."

Scene 4 — Performance results dashboard1:35 – 2:20

On Screen

Performance results page: timeline chart showing p95 response time across 10 runs. One run is highlighted in amber (threshold breach). Metrics table below showing each endpoint with current vs. baseline vs. threshold. Lighthouse score card with colour-coded scores.

Voiceover

"The performance dashboard shows every metric across runs — making it easy to see trends, not just snapshots. When a threshold is breached, the run is highlighted and the specific metric flagged. You can see exactly which deploy introduced the regression by looking at the timeline. Click through to see the full Lighthouse report for any run."

Scene 5 — Load test preview2:20 – 3:00

On Screen

Show a load test configuration: concurrent users (10, 50, 100), ramp-up period, test duration. Show results: throughput chart, error rate line, response time chart showing degradation under load.

Voiceover

"For APIs that need to handle concurrent traffic, VigilQA includes a lightweight load test: configure the concurrency profile — ramp from 10 to 100 users over 60 seconds — and VigilQA measures how throughput, error rate, and response time degrade under load. Know your breaking point before your users find it."
