TalentDesk Docs

Common Issues

Quick fixes for the issues teams hit most often when setting up and running TalentDesk.

Cannot mark an engineer as Deputed

The Mark as Deputed button stays disabled until all four Placement Quality Gate checkpoints are ticked. Open the deployment's Placement Gate tab and confirm that pre-screen, technical bar, background check, and client alignment are all checked. Any unchecked item blocks deputation. See Deployments & Placement Gate.

An Account Manager cannot see a client

Account Managers only see clients assigned to them. Confirm the client's assigned Account Manager under the client detail page. A Master Admin can reassign the client under Settings → Users or on the client record. See Roles & Access.

A placement keeps appearing under pending health updates

  • The Action Center flags any Deputed deployment with no health signal in the past 7 days. Log this week's Green / Amber / Red signal on the deployment's Health tab to clear it.
  • Amber and Red signals require a note — the signal will not save without one.

Cannot save an Amber or Red health signal

A note is mandatory for non-green signals. Add a short explanation of the concern before saving. Only Green signals can be logged without a note.

Cannot add a new skill to a position or profile

Only Master Admins can add skills to the shared skills library. Account Managers and Recruiters can use existing skills but cannot create new ones. Ask a Master Admin to add the skill under Settings → Skills.

Database or first-run issues

On first launch, TalentDesk creates a SQLite database at data/talentdesk.db and seeds a Master Admin account whose credentials are printed to the terminal. If login fails, confirm the database file was created and check the startup logs for the seed credentials.
