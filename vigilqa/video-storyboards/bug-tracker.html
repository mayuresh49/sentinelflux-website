Bug Tracker

Auto-created, evidence-rich bug entries from test failures

~3:00
estimated duration
Target: QA engineers, dev teams
Scene 1 — Hook0:00 – 0:15

On Screen

Timer: QA engineer copying traceback from CI, opening JIRA, pasting screenshot, writing description, tagging severity, assigning — each step adds to a running "5 minutes per bug" counter.

Voiceover

"Every test failure requires a bug report. Traceback, screenshot, steps to reproduce, severity, assignee. It takes five minutes per failure, and nobody enjoys it. VigilQA's Bug Tracker module creates fully evidenced bug entries automatically — from test failure to tracked bug in seconds."

Scene 2 — Auto-created bug entry0:15 – 1:00

On Screen

Navigate to Bugs. Show the automatically created bug entry. Fields visible: title (derived from test name + failure message), severity (auto-classified by ResultAnalyzer), module, test run link, status (Open). Click to expand the full bug detail.

Voiceover

"Every time a test fails and ResultAnalyzer classifies it as a real bug — not a flaky test, not a locator issue — a bug entry is created automatically. The title, severity, and module are set from the failure analysis. No manual triage required."

Scene 3 — Bug detail with evidence1:00 – 1:45

On Screen

Bug detail page: screenshot at failure point, traceback, step log up to failure, the expected vs. actual assertion, link to the Playwright trace, link to the test run. All pre-populated.

Voiceover

"The bug detail page has everything needed to reproduce and fix the issue — screenshot at the moment of failure, full traceback, step-by-step log showing where it diverged from expected behaviour, and a link to the Playwright trace for deep investigation. Everything the engineer needs is already there."

Scene 4 — Workflow: triage, assign, resolve1:45 – 2:20

On Screen

Bug list: change status to In Progress. Add a comment. Change assignee. Then show a "Mark Resolved" button — click it, status changes to Resolved with the resolving test run linked.

Voiceover

"The workflow is simple: Open → In Progress → Resolved. Add comments, assign to a team member, and when the fix ships and the test passes on the next run, the bug is automatically linked to the resolving run. You have a complete audit trail — from first failure to passing test — without touching a separate bug tracker."
