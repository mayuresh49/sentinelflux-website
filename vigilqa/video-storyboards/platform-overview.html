Scene 2 — What is VigilQA 0:20 – 0:50

On Screen

Animated diagram: Knowledge Base → Generation Pipeline → Test Execution → Post-Run Agents → loop back

Each node highlights as it's mentioned

Voiceover

"VigilQA is an AI-powered test automation platform that works in four continuous phases. You describe your application in a structured Knowledge Base. AI agents generate test scripts from that description. Tests run on every commit. And after each run, five more agents automatically classify failures, detect regressions, measure coverage, and heal broken selectors — without you touching a line of test code."

Scene 4 — Knowledge Base 1:30 – 2:15

On Screen

Navigate to KB editor. Show a domain file open with scenario entries — id, title, steps, expected. Highlight a specific scenario. Then show the "Extract from Document" button.

Voiceover

"The Knowledge Base is where you tell VigilQA what your application does. Instead of writing test code, you write plain descriptions of user flows — what the user does, what the system should respond with. Each entry becomes the input for AI-generated test scripts. If you already have specs, user stories, or onboarding docs, Extract from Document can convert them into KB entries in minutes."

Scene 5 — Test Plans and Runs 2:15 – 3:00

On Screen

Navigate to Test Plans. Show a plan with modules enabled (checkboxes: Web UI ✓, API ✓, Visual ✓). Click "Run Now". Cut to the run progress view showing live test output. Then cut to the completed run summary.

Voiceover

"Test Plans let you define exactly which modules to run and against which environment. You can trigger a run manually from here, or set it to fire automatically on every git push via CI/CD. Each run produces a detailed result — passed, failed, and skipped, with full traces for every failure."

Scene 6 — Approvals workflow 3:00 – 3:40

On Screen

Navigate to Approvals. Show a locator heal pending approval — before/after selectors highlighted. Click Approve. Show a script patch pending approval — diff view. Click Approve.

Voiceover

"Every AI-generated change goes through an approval queue before it's committed. This isn't a limitation — it's a design principle. AI suggests, humans decide. Locator heal proposals, script patches, new test cases — all reviewable before they go live. You stay in control of what changes and when."

Scene 7 — CI/CD integration 3:40 – 4:20

Voiceover
