On Screen

Voiceover

"If you've ever spent a morning fixing test selectors instead of finding actual bugs, you know the problem. Playwright tests are powerful, but they're brittle — and the maintenance cost compounds every time your UI changes. VigilQA's Web UI module eliminates that maintenance burden."

Scene 2 — How it works0:20 – 0:50

On Screen

Diagram: KB scenario → AppExplorer crawls app → ScriptGen writes Playwright test → test runs → LocatorHealer monitors

Voiceover

"You describe a user flow in the Knowledge Base — what the user does, what they should see. AppExplorer crawls your live application to capture real DOM selectors. ScriptGen writes a Playwright test using those selectors. And when the UI changes, LocatorHealer automatically detects broken selectors and proposes replacements — you just approve them."

Voiceover

"Here's a Web UI scenario in the Knowledge Base — a checkout flow. We describe the steps in plain language: add item to cart, proceed to checkout, enter payment details, confirm order. That's it. VigilQA translates this into a complete Playwright test, with real selectors captured from your live app."

Scene 4 — Generated script1:30 – 2:10

On Screen

Navigate to Scripts view. Show the generated test_chk001.py file. Scroll through it — visible: real locators, Playwright page.click() and page.fill() calls, assertions. Highlight the data-testid selectors.

Voiceover

"This is the generated Playwright test. Notice it uses data-testid selectors — the most stable type. VigilQA prioritises these over CSS classes or XPath, which means your tests survive most UI refactors without any changes. The script is human-readable and editable — you own it."

On Screen

Trigger a test run. Cut to run progress view — Web UI tests executing, progress bar. Cut to completed run — 12 passed, 1 failed. Click the failure to see: screenshot, Playwright trace link, step-by-step log.

Voiceover

"When a run completes, VigilQA shows you exactly what passed and what didn't. For every failure, you get a screenshot at the moment of failure, the full Playwright trace, and a step-by-step log. The ResultAnalyzer agent has already classified the failure — is it a real bug, a locator issue, or a flaky test? You see that classification before you even open the trace."

Scene 6 — LocatorHealer in action2:50 – 3:30

On Screen

Navigate to Approvals → Locator Heals tab. Show a pending heal: old selector (strikethrough), new selector proposed by LocatorHealer. Show the confidence score and how it was discovered. Click Approve.

Voiceover

"When a locator breaks, LocatorHealer automatically discovers replacement selectors — first from stored alternatives, then by re-crawling the app, and finally by asking an AI to suggest new ones. Each proposal appears here for your approval, with a confidence score and the reasoning behind it. One click to approve and the test is back to green."
