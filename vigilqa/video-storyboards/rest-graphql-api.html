On Screen

KB editor open. Show an API scenario: endpoint, method, steps, expected response fields and status code. Highlight the module: api field. Show a second scenario for an auth flow.

Voiceover

"API scenarios in the Knowledge Base describe what you expect from each endpoint — the method, the request structure, and what the response should contain. You can describe happy paths, error cases, auth flows, and pagination. No JSON schema required — plain English descriptions are enough for the generation agents to produce accurate assertions."

Scene 3 — Generated test1:00 – 1:45

On Screen

Scripts view: open a generated API test file. Visible: requests.post() call, auth header injection, assert response.status_code == 201, field-level assertions on the response JSON. Scroll through to show the full coverage.

Voiceover

"The generated test uses Python requests. It sends the actual HTTP call, injects auth credentials from your environment config, and asserts on status code, response body structure, and field values. ScriptReview agent checks the generated test for correctness before it's committed — you review the output, not debug it."

Scene 4 — GraphQL support1:45 – 2:20

On Screen

Show a KB scenario with module: api and endpoint: /graphql. Switch to the generated test — shows a GraphQL query string being POSTed, response data extraction, and assertion on nested fields.

Voiceover

"GraphQL endpoints work the same way. Describe the query, the variables, and the expected shape of the response. VigilQA generates a test that POSTs the GraphQL operation and validates the nested response fields. Error handling for partial data and GraphQL errors is included automatically."

Scene 5 — Run results and failure analysis2:20 – 3:10

On Screen

Completed test run view. Click a failed API test. Show: the request sent (method, headers, body), the actual response received, the assertion that failed with expected vs. actual values highlighted in red/green.

Voiceover

"When an API test fails, VigilQA shows you the full request-response pair — exactly what was sent and exactly what came back. The assertion failure highlights the expected value versus what the API actually returned. No log digging, no reproducing in Postman. The evidence is right here."
